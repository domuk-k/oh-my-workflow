Quickstart

Add one command to your coding agent.

Install the authoring skill, then ask your own agent to create and run a workflow. omw defaults to --agent auto, so the skill can call the CLI without making you pick Claude, Codex, or Hermes. 

npx skills add domuk-k/oh-my-workflow --skill omw

Why now

Agent CLIs made workflow glue the bottleneck.

Coding agents are no longer only chat panes. They are CLIs you can launch, resume, bound, and compose. Most teams still rebuild the same orchestration shell each time.

Closed workflow surfaces proved the pattern

Native dynamic workflows are useful. omw makes the authoring shape portable across hosts.

Schema gates turn prose into contracts

Each node can self-repair against JSON Schema errors before your script sees the result.

Journals make agent work debuggable

Every phase, attempt, failure kind, cached hit, and final result is replayable as JSONL.

API

The whole runtime is seven hooks.

No DSL, no source transform, no ambient globals. Destructure only what you use.

agent

Run one coding-agent CLI node. Returns validated output, raw text, or null.

parallel

Run independent thunks concurrently and keep failures as null.

pipeline

Move each item through stages independently without stage-wide barriers.

workflow

Run one nested workflow, sharing adapter, journal, and budget pool.

budget

Read total, spent, and remaining output-token budget.

phase

Group later calls under a journal and pretty-tree heading.

log

Write narration into the side-channel journal, never stdout.

Patterns

Use it when the graph matters.

omw is for multi-step agent work: search, verify, vote, synthesize, loop, resume.

Fan-out

Launch N research or review nodes with bounded concurrency, then dedupe and rank survivors.

Verify-vote

Have independent agents judge a finding. Failed nodes abstain instead of voting no.

Loop-until-dry

Keep asking for one more issue until a round returns no new validated result.

Launch proof

Small enough to audit, tested enough to trust.

The repo ships a conformance suite, live adapter smoke tests behind OMW_LIVE=1, Vercel static docs configuration, and a Show HN launch note in docs/launch/.
